1. Как вы можете описать процесс распространения событий в
JavaScript (Event Propagation)? Расскажите о его основных фазах.
Какие есть способы применения знаний о фазах
рааспространения событий на практике?


Event Propagation (распространение событий) в JavaScript — это процесс, по которому браузер определяет, какие элементы получат событие (например, клик) и в каком порядке.

Событие проходит три основные фазы:
- Capturing Phase (фаза перехвата): событие идет сверху вниз по дереву DOM, начиная с window и проходя через все родительские элементы до целевого.
- Target Phase (фаза цели): событие достигает самого элемента, на котором оно произошло.
- Bubbling Phase (фаза всплытия): событие поднимается снизу вверх, проходя через родителей к window.

По умолчанию большинство событий во фронтенде используют всплытие (bubbling).

Применение на практике:
- Можно повесить один обработчик на родителя вместо каждого дочернего элемента. Например, для списка <ul> можно обработать клик на любом <li> через event delegation.
- Можно остановить всплытие с помощью event.stopPropagation() если нужно, чтобы событие не доходило до родителей.
- Можно обрабатывать событие на фазе захвата, передав true третьим параметром в addEventListener.


2. Расскажите, что такое Promise в JavaScript, как он работает, какие
основные состояния у него бывают? Каким еще способом можно
обработать асинхронный код? Какую роль в асинхронности играет
событийный цикл (Event Loop)?

Promise – это объект, который представляет результат успешного выполнения или ошибку асинхронной операции. Они позволяют работать с асинхроным кодом, управлять последовательностью операций и обрабатывать результаты этих операций.

Состояния Promise:
- pending – ожидание выполнения.
- fulfilled – операция завершена успешно.
- rejected – операция завершилась с ошибкой.

Другие способы работы с асинхронным кодом:
- Колбэки (callback) – старый способ, но могут возникнуть "callback hell".
- async/await – современный синтаксис поверх Promise, позволяет писать код как синхронный.

Event Loop и асинхронность:
Event Loop позволяет JavaScript выполнять задачи асинхронно, не блокируя выполнение другого кода.

3. Что такое ООП, и какие его ключевые принципы вы могли бы
описать? При объяснении приводите примеры. Максимально
подробно расскажите о том, как ООП реализовано в JavaScript.

ООП (Объектно-Ориентированное Программирование) – это подход к программированию, где вместо представлено в виде объектов, которые содержат данные (свойства) и поведение (методы).

Ключевые принципы ООП:
- Encapsulation (инкапсуляция) – скрытие внутренней реализации, доступ через методы.
- Abstraction (абстракция) – выделение важного, сокрытие деталей.
- Inheritance (наследование) – создание новых объектов на основе существующих.
- Polymorphism (полиморфизм) – объекты могут иметь одинаковый интерфейс, но разное поведение.

В JavaScript ООП можно реализовать через классы, функции-конструкторы, а также через прототипное наследование.

4. Опишите максимально детально, как браузер обрабатывает URL,
начиная с момента его ввода в адресную строку и заканчивая
отображением страницы. Какие технологии позволяют ускорить
процесс? Какие проблемы (безопастность, междоменное
взаимодействие) могут возникать в процессе?

Когда вводишь URL в адресную строку, браузер проходит несколько шагов:
- DNS Lookup – браузер ищет IP-адрес сервера по доменному имени.
- TCP Connection – устанавливается соединение с сервером через протокол HTTP/HTTPS.
- Отправка запроса – браузер отправляет HTTP-запрос на сервер.
- Ответ сервера – сервер возвращает HTML, CSS, JS, картинки, видео и т.д.
- Парсинг HTML – браузер строит DOM-дерево.
- Парсинг CSS – строится CSSOM.
- Рендеринг – браузер объединяет DOM и CSSOM в Render Tree, вычисляет layout и отображает страницу.
- Выполнение JS – браузер выполняет скрипты, которые могут модифицировать DOM.

Технологии для ускорения:
- Кэширование – повторные запросы быстрее.
- CDN – доставляет контент ближе к пользователю.
- Минификация и сжатие – уменьшают размер файлов.
- Lazy loading – загружает ресурсы по мере необходимости.

Проблемы:
- Безопасность – XSS, CSRF, поддельные сертификаты.
- CORS – ограничения на междоменные запросы.
